# BlackChain Internal Launch Dashboard Schema

Use this checklist as the minimum internal launch dashboard schema for the current launch path.

## Launch Record Fields

- launch date
- customer or site
- host-site identifier
- release identifier
- plan tier
- issued hardware tier
- operator owner
- support owner
- activation status
- current incident status

## Verification Fields

- services active: YES / NO
- listeners present: YES / NO
- HTTPS health verified: YES / NO
- HTTPS readiness verified: YES / NO
- `blackctl` trust verified: YES / NO
- finality snapshot verified: YES / NO
- rollback ready: YES / NO

## Daily Tracking Fields

- latest ops log reference
- latest field verification reference
- latest support case reference
- open issues count
- SEV-1 count
- SEV-2 count
- SEV-3 count

## Checklist

- customer onboarding sheet completed
- activation checklist completed
- host-site onboarding sheet completed
- operator signoff completed
- first 72-hour monitoring assigned
