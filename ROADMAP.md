BlackChain Protocol Roadmap

Phase 1 — Chain Integrity

• Block signatures
• Wallet keypairs
• Transaction signatures
• Signature verification before mempool acceptance

Phase 2 — State Structure

• Merkle balance tree
• Deterministic state hashing
• Merkle root stored in block header

Phase 3 — Consensus

• Validator identity
• Signed validator votes
• Commit threshold
• Block finalization

Phase 4 — Sync Hardening

• Deterministic snapshot generation
• Snapshot verification
• Fast sync bootstrapping

Phase 5 — Economics

• Utility Notes mint rules
• Transaction fees
• Inflation schedule

Phase 6 — Security

• Peer authentication
• Encrypted transport
• Rate limiting

BlackChain is designed as a decentralized mesh-based blockchain capable of running on edge devices and distributed networks.

The protocol is being developed and maintained by BlackChain Labs LLC.
