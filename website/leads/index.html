Lead Workflow

BlackChain Website Lead Workflow

One compact static surface for the current website lead path: handoff, first response, qualification, tracking, and territory-pipeline handoff.

New

  • Confirm the website submit path reached `/territory/thanks/`.
  • Copy the captured record using the documented field names.
  • Place it in the flat-file lead store.

Contacted

  • Send the first response.
  • Restate the territory or town.
  • Record the next follow-up date.

Reviewing

  • Check host-site and operator interest.
  • Review plan tier context.
  • Add review notes before qualification.

Qualified

  • Assign an owner.
  • Set the next action.
  • Prepare territory pipeline handoff.

Waiting

  • Track missing details or delayed reply.
  • Keep the next action visible.
  • Review for stale follow-up.

Closed

  • Mark the record clearly.
  • Keep the historical note brief.
  • Do not delete the original lead context prematurely.
