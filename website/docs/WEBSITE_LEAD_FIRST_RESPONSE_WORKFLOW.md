# Website Lead First-Response Workflow

## Goal

- give new website leads a first operational response without changing the static site

## Workflow

1. Review the captured territory submission.
2. Confirm the contact fields are usable.
3. Confirm whether the lead is primarily territory, host-site, or operator oriented.
4. Mark the lead status as `new`.
5. Prepare the first reply using the current website and outreach docs.
6. Send the first response.
7. Update the lead status to `contacted`.
8. Record any follow-up date in the status tracker.

## First-Response Focus

- confirm receipt
- restate the territory or town named by the lead
- acknowledge any host-site or operator interest
- point the lead to the next relevant path:
  - territory review
  - host-site conversation
  - operator conversation

## Notes

- keep the first response aligned to current launch-path docs
- do not promise workflow behavior that does not exist in the static site
