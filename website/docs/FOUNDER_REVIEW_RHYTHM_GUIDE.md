# Founder Review Rhythm Guide

## Daily

- update the founder daily dashboard
- review new website leads and follow-ups due today
- review territory movement and current launch blockers
- update today's priorities

## Weekly

- run the weekly launch review
- compare lead progress, territory progress, and launch progress
- review the decision queue and close stale items

## As Needed

- record major decisions in the decision queue
- update the territory pipeline after qualification changes
- refresh launch and ops references if a blocker or incident changes direction

## Notes

- use this guide with the existing daily cadence and weekly launch review docs
- keep the founder cockpit as a thin operating layer, not a separate system
